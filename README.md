
# PP02 — Производственная практика (Металлургическое производство)

## Описание
---
В ходе практики выполнялись задачи по проектированию и разработке элементов информационной системы для автоматизации учета плавок, контроля качества стали, движения шихтовых материалов и готовой продукции.

Практика выполнялась на базе ООО «УралМеталлСтрой» (отдел АСУ ТП) с использованием **SQL (PostgreSQL)**, **PlantUML** и средств разработки баз данных.

---

## Содержание репозитория

### `docs/`
Документация по практике:
- Отчёт по производственной практике
- Описание предметной области (металлургическое производство)
- Техническое задание
- UML-диаграммы (Use Case, Class, Sequence, Activity, State, Deployment)

### `scripts/`
SQL-скрипты для автоматизации учета:
- `create_db.sql` — создание структуры базы данных (таблицы: плавки, материалы, заказы, сертификаты)
- `add_customer.sql` — добавление заказчика металлопродукции
- `create_contract.sql` — создание договора поставки
- `create_melting_order.sql` — создание заказа-наряда на плавку
- `add_material_request.sql` — заявка на шихтовые материалы
- `create_quality_certificate.sql` — формирование сертификата качества (паспорт плавки)
- `write_off_materials.sql` — списание материалов на производство
- `reports.sql` — отчеты по себестоимости, выходу годного и остаткам

---
## Цель работы

Целью производственной практики являлось получение практических навыков проектирования информационных систем, автоматизация учета производственных процессов металлургического предприятия, обеспечение прослеживаемости качества продукции и повышение эффективности управления материальными потоками.

---

## Используемые технологии

- **PostgreSQL / SQL** — разработка базы данных
- **PlantUML** — моделирование (диаграммы)
- **Draw.io** — вспомогательная визуализация
- **Git / GitHub** — управление версиями
- **Windows / Linux** — среда разработки

## Результаты

В ходе практики:
- изучены бизнес-процессы металлургического производства (сталеплавильный передел);
- разработана структура базы данных для учета плавок, материалов, заказов и сертификатов;
- создан комплект SQL-скриптов для автоматизации основных операций (заказ плавки, списание материалов, формирование сертификата качества);

- создан публичный репозиторий с результатами работы.
